Git 常用指令总结

Posted on 2018-04-03 | In tools

  1. 配置命令

  • 配置 Git 的姓名和Email地址（自报家门）
  • git config 命令的 –global 参数，表示是针对当前用户起作用，如果不加，那只针对当前仓库起作用
  • 让 Git 显示颜色
  • Git 命令别名设置

  1. 基本操作指令

  • 把当前目录变成 Git 可以管理的仓库
  • 把目录中的文件提交到仓库的暂存区
  • 把暂存区的文件提交到仓库中
  • -m 后面输入的是本次提交的说明，可以输入任意内容，方便从历史记录中找到改动记录
  • 命令返回工作区当前的状态
  • 顾名思义就是查看 difference，显示的格式正式 Unix 通用的 diff 格式
  • 前者是比较工作区和暂存区中的文件；后者是比较工作区和版本库里面最新版本的区别
  • git log 命令显示从最近到最远的提交日志
  • –pretty=oneline 以 版本号和提交说明的简要形式显示日志
  • 用来记录用户的每一次命令
  • 在 Git 中，用 HEAD 表示当前版本，上一个版本就是 HEAD^（或 HEAD~1），上上个版本就是 HEAD^^（或 HEAD~2）。
  • –hard 参数的意义 ？

  1. 撤销修改指令

  • 把文件在工作区的修改全部撤销（修改工作区）
  • 把暂存区的修改撤销（unstage)，重新放回工作区（修改暂存区）

  1. 删除文件指令

  • 工作区操作指令，会造成工作区和版本库的不一致性。存在两种处理方法
  • 处理方法一，确实从版本库中删除该文件
  • 处理方法二，还原
  • git checkout 其实是用版本库里的版本替换工作区的版本，无论工作区是修改还是删除，都不可以”一键还原“

  1. 远程仓库指令

  • 创建 SSH Key
  • 把一个已有的本地仓库同GitHub上的仓库相关联
  • 远程库的默认名字是 origin, 也可以改成其他
  • 由于远程库是空的，我们第一次推送 master 分支时，加上-u 参数，Git 不但会把本地的 master 分支内容推送给远程新的 master 分支，还会把本地的 master 分支和远程的 master 分支关联起来，以后推送或拉取时可以简化命令。
  • 从远程库克隆指令
  • 查看远程库的信息
  • 抓取最新提交，然后在本地合并，解决冲突，再推送

  1. 分支操作指令

  • git checkout 命令加上 -b 参数表示创建并切换
  • git branch 命令会列出所有分支，当前分支前面会标一个 * 号
  • git merge 用于合并指定分支到当前分支
  • 删除指定分支
  • 查看分支的合并情况
  • 合并分支时，加上 –no-ff 参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而 fast forward 合并就看不出来曾经做过合并
  • stash 功能，实现当前工作现场的”储藏“和恢复
  • 强行删除分支
  • 本地创建和远程分支对应的分支
  • 建立本地分支和远程分支的关联

  1. 标签管理指令

Git 相关名词概念解释

Posted on 2018-04-03 | In tools

  1. 版本控制

    能够实现自动记录文件的改动，并且允许同事协同编辑的软件。

    ​

  2. 集中式版本控制系统

    版本库是集中存放在中央服务器的，每次工作时，通过从中央服务器下载最新的版本，工作结束后，将作业推送给中央服务器。集中式版本控制系统的最大劣势是必须联网才能工作。

    ​

  3. 分布式版本控制系统

    分布式版本控制系统没有”中央处理器”， 每个人的电脑上都是一个完整的版本库，工作时没有必要的联网要求。多人协作时，需要将各自的修改推送给对方即可。（为了便利“交换”修改， 通常有一台充当“中央服务器”的电脑，但它不是必要的）。

    ​

  4. 版本库 （Repository)

    版本库又名仓库，英文名 repository . 相当于一个目录，这个目录中的所有文件都可以被 Git 管理起来，每个文件的修改，删除，Git 都能够追踪，以便任何时候都可以追踪历史，或者在将来某个时刻可以”还原“。

    ​

  5. 工作区（Working Directory）

    本地工作的电脑中能够看到的目录；在目录中曾经运行过 git init 命令。

    ​
